# Sample Return Policy

> This demo policy is written for the Enterprise Multi-Tool Agent Platform. It is not a real merchant policy.

## 1. Returnable Conditions

An order may be eligible for return when the request is submitted within 30 days after delivery, the item is unused or has only been opened for inspection, the product category allows returns, the customer can provide order ID and basic issue description, and no previous refund has been completed for the same item.

## 2. Non-Returnable Conditions

Returns may be rejected when the item is food, health, or personal care and has been opened, the order is final sale, the product is damaged by misuse, the request is outside the return window, or the customer cannot provide enough information to identify the order.

## 3. Refund Time

Refunds are normally completed within 3 to 7 business days after the returned item is received and checked. Payment provider delays may extend the final settlement time.

## 4. Abnormal Order Handling

An abnormal order is an order with cancellation, unavailable inventory, delivery delay, payment issue, low review score, refund request, damaged product, wrong item, or customer complaint.

Abnormal orders should be tagged and included in operational reports. The Agent may summarize abnormal order patterns but must not expose sensitive customer data.

## 5. Manual Review Conditions

Manual review is required when the refund amount is higher than the configured threshold, the customer has multiple refund requests in a short period, the issue includes safety or regulatory risk, the AI Agent prepares an external email draft, or SQL query results may include sensitive fields.

## 6. Communication Rules

Customer messages should be clear, polite, and based on verified policy. AI-generated drafts must be reviewed before sending.
